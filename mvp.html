Project Overview

This guide outlines the Minimum Viable Product (MVP) for Shikshak, an AI-powered platform designed to assist teachers by automating assessments, providing deep insights into student learning gaps, and offering actionable performance dashboards.

Core Architecture

Shikshak uses a scalable, decoupled architecture. A Next.js frontend communicates with a FastAPI backend via a REST API. The backend manages business logic, database operations (via SQLAlchemy), and integrates with external AI models from Hugging Face.

Grading Logic Flow

When a student submits a subjective answer, the backend triggers an asynchronous process that classifies the answer without needing a pre-trained model for the specific question.

1. Context Assembly: The student's answer and the teacher's grading rules are combined into a single text sequence.
2. API Call: This sequence is sent to the Hugging Face API along with candidate labels: ["conceptual error", "interpretational error", "procedural error", "correct"].
3. Analysis & Storage: The model returns scores for each label. The highest-scoring label is identified as the error type, and this analysis is saved to the database, ready for teacher review.
